---
title: ソフトウェア工学 第03講 ソフトウェア開発プロセス
---

| id | question | answer | explanation |
|----|----------|--------|-------------|
| se03-001 | ソフトウェアが生まれてから廃棄されるまでの過程を何というか。 | ライフサイクル | 発生・成長・運用・保守を含む。 |
| se03-002 | ソフトウェア開発の手順を一般化したものを何というか。 | プロセスモデル | 開発の流れを表す。 |
| se03-003 | 要求獲得、分析、設計、実装、テストを順に進めるモデルは何か。 | ウォーターフォールモデル | 工程を分けて順番に進める。 |
| se03-004 | 変化する要求に対応し、短い反復を重ねる開発モデルは何か。 | アジャイルソフトウェア開発 | 反復的・適応的に開発する。 |
| se03-005 | リスクを検討しながら反復的に開発するモデルは何か。 | スパイラルモデル | 各反復でリスクを評価する。 |
| se03-006 | 顧客のニーズを引き出す工程を何というか。 | 要求獲得 | requirements elicitation。 |
| se03-007 | 分析工程では、要求仕様書の何を明確にするか。 | 要求事項 | 抜けや漏れを吟味する。 |
| se03-008 | 設計工程で決めるのは「何を作るか」と「どう作るか」のどちらか。 | どう作るか | Howを決める工程。 |
| se03-009 | ソフトウェア全体の基本構造を決める設計を何というか。 | システム設計 | 概要設計・基本設計ともいう。 |
| se03-010 | 個々のプログラムを設計する工程を何というか。 | プログラム設計 | 詳細設計ともいう。 |
| se03-011 | 1つの部品・関数などを対象にするテストは何か。 | 単体テスト | unit test。 |
| se03-012 | ソフトウェアは最初のリリース後も何が必要か。 | 保守・発展・運用 | バグ修正や機能追加を行う。 |
| se03-013 | ウォーターフォールモデルで最初に行う上流工程は何か。 | 要求獲得 | 顧客ニーズを整理する。 |
| se03-014 | システム全体の基本構造を決める工程は何か。 | システム設計 | 概要設計ともいう。 |
| se03-015 | 個々のプログラムを設計する工程は何か。 | プログラム設計 | 詳細設計ともいう。 |
| se03-016 | 実装後に正しさを確認する工程は何か。 | テスト | 欠陥を発見する。 |
| se03-017 | ソフトウェアを実際に利用する段階は何か。 | 運用 | リリース後の利用段階。 |
| se03-018 | 要求仕様書の抜けや漏れを確認する工程は何か。 | 分析 | 結果を要求仕様書へ反映する。 |
| se03-019 | 開発をリスク検討と反復で進めるモデルは何か。 | スパイラルモデル | リスク駆動の反復型。 |
