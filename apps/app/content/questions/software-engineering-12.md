---
title: ソフトウェア工学 第12講 ソフトウェア品質
---

| id | question | answer | explanation |
|----|----------|--------|-------------|
| se12-001 | 製品そのものの品質を何というか。 | プロダクト品質 | バグや使いやすさなど。 |
| se12-002 | 開発の進め方・作り方の品質を何というか。 | プロセス品質 | 生産者への信頼にも関係する。 |
| se12-003 | 利用者から見える性能や信頼性などの品質を何というか。 | 外部品質 | 実行時の品質。 |
| se12-004 | ソースコードの構造や保守性など、内部の作りに関する品質を何というか。 | 内部品質 | 開発成果物の内部的な側面。 |
| se12-005 | テスト実施に伴う累積バグ数の変化を表す曲線を何というか。 | ソフトウェア信頼度成長曲線 | バグ成長曲線ともいう。 |
| se12-006 | 品質特性のうち、要求を満たしているかを表す代表的な特性は何か。 | 機能性 | functionality。 |
| se12-007 | 「ないと不満だが、あって当然」とされる品質を何というか。 | 当たり前品質 | 充足されると不満がなくなる。 |
| se12-008 | なくても不満はないが、あると満足を高める品質を何というか。 | 魅力品質 | 付加価値になる。 |
| se12-009 | Goal・Question・Metricsの略で、測定項目を定める枠組みは何か。 | GQM | 目標から質問とメトリクスを導く。 |
| se12-010 | 成果物の大きさや複雑さを測る指標を何というか。 | プロダクトメトリクス | 製品に関する測定値。 |
| se12-011 | 開発期間や欠陥数など開発活動を測る指標を何というか。 | プロセスメトリクス | プロセスに関する測定値。 |
| se12-012 | 品質を利用者から見た品質と内部の作りに分けると、前者は何か。 | 外部品質 | 実行時に観測される。 |
| se12-013 | ソースコードの行数や複雑さは何メトリクスか。 | プロダクトメトリクス | 製品を測定する。 |
| se12-014 | 開発工数や期間は何メトリクスか。 | プロセスメトリクス | 開発活動を測定する。 |
| se12-015 | レビューの主な目的は何か。 | 欠陥の早期発見 | 後工程の修正コストを下げる。 |
| se12-016 | Goal・Question・Metricsで最初に定めるものは何か。 | Goal | 目標から質問と指標を導く。 |
| se12-017 | テストで累積バグ数の増加が止まることは何の判断材料か。 | テスト終了時期 | 企業ごとに基準は異なる。 |
| se12-018 | 開発履歴から知見を得る活動は何か。 | ソフトウェアリポジトリマイニング | 開発データを分析する。 |
