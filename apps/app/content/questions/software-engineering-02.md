title: ソフトウェア工学 第02講 オブジェクト指向
---

| id | question | answer | explanation |
|----|----------|--------|-------------|
| se02-001 | オブジェクト指向とは、何を中心に考えるプログラミング技術か。 | モノ | オブジェクトを中心に整理する。 |
| se02-002 | オブジェクトが持つ、状態や性格を表すものは何か。 | 属性 | attribute。 |
| se02-003 | オブジェクトが行える動作を何というか。 | 操作 | operation。 |
| se02-004 | オブジェクトの属性と操作を定義するものは何か。 | クラス | オブジェクトのひな形。 |
| se02-005 | クラスから実行時に作られる個々のオブジェクトを何というか。 | インスタンス | 同じクラスから複数作れる。 |
| se02-006 | オブジェクト指向の三原則を答えよ。 | カプセル化・継承・多態性 | Encapsulation, Inheritance, Polymorphism。 |
| se02-007 | 複雑な内部実装を隠し、外から単純に見せる仕組みは何か。 | カプセル化 | 変更の影響を閉じ込めやすい。 |
| se02-008 | 既存クラスの機能を引き継いで拡張する仕組みは何か。 | 継承 | 共通点をまとめられる。 |
| se02-009 | 受け手のオブジェクトによって動作が定まる性質は何か。 | 多態性 | ポリモーフィズムともいう。 |
| se02-010 | オブジェクト指向の記述法として代表的なモデリング言語は何か。 | UML | Unified Modeling Language。 |
| se02-011 | 具体的なクラスではなく、抽象的なインターフェースに対してプログラムを書く考え方を何というか。 | 抽象に依存する | 具体的なものへの依存を減らす。 |
| se02-012 | 手続き型で中心となるものは何か。 | 処理・関数 | 手続きの流れを中心に考える。 |
| se02-013 | オブジェクトが持つ状態を表すものは何か。 | 属性 | attribute。 |
| se02-014 | オブジェクトが提供する動作を何というか。 | 操作 | methodともいう。 |
| se02-015 | 継承元のクラスを何というか。 | スーパークラス | 基底クラス。 |
| se02-016 | 継承して作られたクラスを何というか。 | サブクラス | 派生クラス。 |
| se02-017 | 複雑な実装を外部から隠すことを何というか。 | 情報隠蔽 | カプセル化と関係する。 |
| se02-018 | 変更に強い設計を目指すために分離するものは何か。 | 変更される部分 | 変更容易性を高める。 |
