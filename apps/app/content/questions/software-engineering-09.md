---
title: ソフトウェア工学 第09講 ソフトウェアテスト
---

| id | question | answer | explanation |
|----|----------|--------|-------------|
| se09-001 | テストの主な目的は、ソフトウェアの何を発見することか。 | 欠陥 | 要求を満たすかも確認する。 |
| se09-002 | テスト対象を実行して結果を確認するテストを何というか。 | 動的テスト | 実行を伴うテスト。 |
| se09-003 | プログラムを実行せずにレビューや解析を行うテストを何というか。 | 静的テスト | ソースコードレビューなど。 |
| se09-004 | 最小単位の部品を個別に検証するテストは何か。 | 単体テスト | unit test。 |
| se09-005 | 複数部品を組み合わせて検証するテストは何か。 | 結合テスト | integration test。 |
| se09-006 | システム全体を検証するテストは何か。 | システムテスト | system test。 |
| se09-007 | 顧客・利用者の要求を満たすか確認するテストは何か。 | 受け入れテスト | acceptance test。 |
| se09-008 | 入力を同値なグループに分け、代表値を選ぶ技法は何か。 | 同値分割 | equivalence partitioning。 |
| se09-009 | 境界付近の値を重点的に調べる技法は何か。 | 境界値分析 | boundary value analysis。 |
| se09-010 | すべてのテストを実施しても欠陥がないと断言できないことを何というか。 | テストの限界 | テストは欠陥の存在を示せても不在は証明しにくい。 |
| se09-011 | プログラムの全ての実行経路を扱うと起きやすい問題は何か。 | パス爆発 | 経路数が急増する。 |
| se09-012 | テストの入力・期待結果をまとめたものは何か。 | テストケース | test case。 |
| se09-013 | 修正後に以前の機能が壊れていないか確認するテストは何か。 | 回帰テスト | regression test。 |
| se09-014 | 負荷を高めて性能の限界を調べるテストは何か。 | ストレステスト | stress test。 |
| se09-015 | 内部構造を考慮するテストは何か。 | ホワイトボックステスト | 内部論理を利用する。 |
| se09-016 | 内部構造を考慮せず入出力を見るテストは何か。 | ブラックボックステスト | 外部仕様を利用する。 |
| se09-017 | 仕様を基にテストデータを作る代表技法は何か。 | 同値分割 | 入力をグループ化する。 |
| se09-018 | 境界付近の値を調べる代表技法は何か。 | 境界値分析 | 境界で起きる欠陥を探す。 |
